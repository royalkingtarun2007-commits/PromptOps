name: Prompt Regression Tests

on:
  pull_request:
    paths: ['prompts/**']

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: your-username/promptops@v1
        with:
          api-key:      ${{ secrets.PROMPTOPS_KEY }}
          suite:         ./promptops.tests.json
          threshold:     0.85
          fail-on-drift: true
Why PromptOps

Prompts deserve
engineering discipline.

Your code is versioned, reviewed, and tested before every deploy. Your prompts — the most critical part of your AI product — are not. Until now.

Git-style version control
Every prompt change tracked with semantic diffs. See not just what changed, but how the meaning shifted. Branch, rollback, and audit with confidence.
A/B testing in production
Split real traffic between prompt versions. Measure performance on actual users. Auto-promote the winner — no redeploy needed.
PR-style review workflow
Product managers write prompts. Engineers review them. Approval gates block untested prompts from reaching production.
Prompt CI — block drift
Define a golden test suite. Run it on every pull request. Block merges when prompt changes break expected outputs beyond your threshold.
Fully provider-agnostic
One SDK. Works with OpenAI, Anthropic, Gemini, Mistral, and Ollama. Switch providers without changing your application code.
Self-host in 60 seconds
One Docker Compose command. Your prompts live on your infrastructure. Zero data leaves your environment. Full audit log built in.
