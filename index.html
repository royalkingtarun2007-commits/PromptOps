# .github/workflows/prompt-ci.yml
name: Prompt Regression Tests

on:
  pull_request:
    paths:
      - 'prompts/**'

jobs:
  test-prompts:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: promptops/action@v1
        with:
          api-key: ${{ secrets.PROMPTOPS_KEY }}
          suite: ./tests/prompts.json
          threshold: 0.85    # semantic similarity
          fail-on-drift: true  # block the PR
// Why PromptOps

Prompts deserve
engineering discipline.

Your code is versioned, reviewed, and tested. Your prompts — the most critical part of your AI product — aren't. Until now.

🔀
Git-style version control
Every prompt change tracked with semantic diffs. See not just what changed, but how the meaning shifted. Branch, merge, and rollback with confidence.
⚖️
A/B testing in production
Split traffic between two prompt versions and measure real performance. Auto-promote the winner and retire the loser — all without a deploy.
👥
PR-style review workflow
PMs write prompts. Engineers review them. Designers see what's live. Built-in approval gates with roles: viewer, editor, reviewer, approver.
🧪
Prompt CI — block bad changes
Define a golden test suite. Run it in CI on every PR. Block deployments when prompt changes break expected outputs beyond your threshold.
🔌
Fully provider-agnostic
One SDK. Works with OpenAI, Anthropic, Gemini, Mistral, and Ollama. Switch providers without changing a line of your application code.
🏠
Self-host in 60 seconds
One Docker Compose command. Your prompts live on your infra. Zero data leaves your environment. Full audit log included.
